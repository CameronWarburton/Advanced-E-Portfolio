  • Netflix Clone

    React, TypeScript, Tailwind CSS, Next.js, Next-Auth, Prisma, Supabase

    I created this project to practise the skills I had learnt throughtout my course. I hadn't used TypeScript or Tailwind before so this really opened my eyes to how both of these worked and how they can make development simpler and more efficient. I used Prisma and Supabase to store the Movies/TV Shows and also paired with Next-Auth to allow users to login to the site and also signup. You can also login via GitHub OAuth but Googles OAuth required me to be an organisation so I was unable to implement this in. The site works on all devices and with tailwind I was able to make the site responsive to various screen sizes right in the classnames of each element. The Next.js layout/temlate was different to what I had used before but it was so much easy to route each page of the site than in just raw React. I am proud of what I was able to achieve within this project and I looking forward to what I will be able to create next.

  • Spotify Clone

    React, Tailwind CSS, Vite

    To build a Spotify clone, I used React for its powerful component-based architecture, which allowed me to efficiently manage the app's state and UI logic. Vite.js served as the build tool and development server, providing a fast and seamless development experience with its optimized hot module replacement (HMR) and lightning-fast startup times. For styling, I chose Tailwind CSS due to its utility-first approach, which enabled me to rapidly prototype and design responsive, modern interfaces without writing a lot of custom CSS. Together, these technologies allowed me to replicate Spotify's sleek and interactive user interface, incorporating features like music playback and real-time updates with a streamlined and efficient workflow.
